# SmartEnv-Gateway SE-100 产品概述

## 1. 文档说明

本文档用于介绍 SmartEnv-Gateway SE-100 智能环境监测网关的基本定位、主要功能、联网方式、指示灯含义和使用边界。

SE-100 是一个受控虚拟产品，用于构建 Personal Knowledge Assistant 项目的技术支持知识库。后续系统回答用户问题时，只应依据本知识库中的产品说明、故障手册、售后 FAQ 和历史工单案例进行回答。

本文档不用于解释真实硬件电路、底层驱动开发或云平台源码实现。

---

## 2. 产品定位

SmartEnv-Gateway SE-100 是一款面向小型办公室、实验室、仓储间和设备间的智能环境监测网关。

SE-100 主要用于采集环境温湿度、门磁状态和外接传感器数据，并通过网络将数据上报到云端平台。用户可以通过本地 Web 配置页完成联网配置、设备绑定、MQTT 参数配置和基础维护操作。

SE-100 的目标用户包括：

* 小型企业 IT 运维人员
* 实验室管理员
* 仓储环境管理人员
* 售后技术支持人员
* 需要远程查看环境状态的普通用户

---

## 3. 核心功能

SE-100 提供以下核心功能：

| 功能模块      | 说明                      |
| --------- | ----------------------- |
| 环境数据采集    | 采集温度、湿度、门磁状态和外接传感器数据    |
| 网络连接      | 支持以太网和 Wi-Fi 连接         |
| MQTT 数据上报 | 将设备状态和传感器数据上报到云端平台      |
| 本地 Web 配置 | 通过浏览器进入本地配置页，修改联网和平台参数  |
| 设备状态指示    | 通过电源灯、网络灯、云端灯和告警灯显示当前状态 |
| OTA 升级    | 支持通过云端下发固件升级任务          |
| 异常告警      | 当温湿度超过阈值或传感器离线时触发告警     |
| 日志导出      | 支持导出最近运行日志，便于售后排查问题     |

## 4. 典型使用场景

### 4.1 办公室环境监测

用户将 SE-100 放置在办公室或会议室中，用于查看温度、湿度和设备在线状态。设备联网成功后，会按照固定周期向云端上报环境数据。

### 4.2 仓储环境监测

用户将 SE-100 放置在仓储间，用于监测温湿度是否超过设定范围。当湿度过高或温度异常时，SE-100 会在本地显示告警状态，并将告警事件上报到云端。

### 4.3 实验室设备间监测

实验室管理员可以通过 SE-100 监测设备间环境变化。如果网络中断，设备会保留最近的本地状态；网络恢复后，设备继续向云端上报最新数据。

### 4.4 售后远程排障

当用户反馈“设备离线”“数据不上报”“无法进入配置页”等问题时，技术支持人员可以根据指示灯状态、网络连接方式、MQTT 配置和运行日志进行排查。

## 5. 设备外观与接口

SE-100 正面包含 4 个状态指示灯：

| 指示灯   | 名称  | 用途              |
| ----- | --- | --------------- |
| PWR   | 电源灯 | 显示设备是否正常供电      |
| NET   | 网络灯 | 显示设备是否连接到局域网    |
| CLOUD | 云端灯 | 显示设备是否成功连接云端平台  |
| ALARM | 告警灯 | 显示是否存在环境异常或设备异常 |

SE-100 侧面包含以下接口：

| 接口     | 说明           |
| ------ | ------------ |
| DC 12V | 电源输入接口       |
| LAN    | 以太网接口        |
| RESET  | 复位按键         |
| USB-C  | 维护接口，仅用于售后调试 |
| SENSOR | 外接传感器接口      |

普通用户不需要使用 USB-C 维护接口。售后人员排障时，可以根据日志导出流程获取设备运行信息。

## 6. 联网方式

SE-100 支持两种联网方式：以太网联网和 Wi-Fi 联网。

### 6.1 以太网联网

用户将网线插入 SE-100 的 LAN 接口后，设备会自动尝试通过 DHCP 获取局域网 IP 地址。

当 NET 指示灯常亮时，表示设备已经连接到局域网。

当 NET 指示灯慢闪时，表示设备正在获取 IP 地址或网络连接不稳定。

### 6.2 Wi-Fi 联网

首次使用或恢复出厂设置后，SE-100 会进入配置模式。用户可以连接设备发出的配置热点，然后进入本地 Web 配置页填写 Wi-Fi 名称和密码。

默认配置热点名称格式为：

```text
SE100-Setup-设备后四位
```

本地 Web 配置页默认地址为：

```text
http://192.168.4.1
```

配置完成并保存后，设备会重启并尝试连接用户填写的 Wi-Fi 网络。

## 7. MQTT 上报说明

SE-100 通过 MQTT 协议向云端平台上报设备状态、传感器数据和告警事件。

常见上报内容包括：

| 数据类型 | 示例内容                 |
| ---- | -------------------- |
| 设备状态 | 在线、离线、重启、配置更新        |
| 环境数据 | 温度、湿度、门磁状态           |
| 告警事件 | 温度超限、湿度超限、传感器离线      |
| 运维信息 | 固件版本、网络信号强度、最近一次上报时间 |

默认情况下，SE-100 每 60 秒上报一次环境数据。发生告警事件时，设备会立即上报告警信息。

如果 CLOUD 指示灯未常亮，通常表示设备未成功连接云端平台。常见原因包括网络不可用、MQTT 地址填写错误、设备密钥错误或云端平台暂时不可达。

## 8. 本地 Web 配置页

用户可以通过本地 Web 配置页完成以下操作：

* 查看设备基本信息
* 配置 Wi-Fi 网络
* 配置 MQTT 服务器地址
* 修改数据上报周期
* 设置温湿度告警阈值
* 查看当前网络状态
* 导出最近运行日志
* 恢复出厂设置
* 检查当前固件版本

本地 Web 配置页主要用于设备首次安装、网络变更和售后排障。普通使用过程中，用户通常不需要频繁进入配置页。

## 9. 指示灯状态总览

| 状态      | PWR 电源灯 | NET 网络灯 | CLOUD 云端灯 | ALARM 告警灯 |
| ------- | ------- | ------- | --------- | --------- |
| 正常运行    | 常亮      | 常亮      | 常亮        | 熄灭        |
| 正在启动    | 常亮      | 慢闪      | 熄灭        | 熄灭        |
| 网络未连接   | 常亮      | 熄灭      | 熄灭        | 熄灭        |
| 已联网但未连云 | 常亮      | 常亮      | 慢闪        | 熄灭        |
| 存在告警    | 常亮      | 常亮      | 常亮或慢闪     | 常亮        |
| 配置模式    | 常亮      | 快闪      | 熄灭        | 慢闪        |
| 固件升级中   | 常亮      | 常亮或慢闪   | 快闪        | 熄灭        |

指示灯状态是售后排障的重要入口。用户反馈问题时，技术支持人员应优先确认 4 个指示灯的当前状态。

## 10. OTA 升级说明

SE-100 支持 OTA 升级。云端平台下发升级任务后，设备会下载固件包并自动完成升级。

升级过程中，CLOUD 指示灯会快闪。升级完成后，设备会自动重启。

升级期间请不要断电。断电可能导致升级失败。若升级失败，设备会保留当前可用版本，并在下一次联网后重新检查升级任务。

用户一般不需要手动上传固件包。OTA 升级任务由平台管理员或售后人员发起。

## 11. 使用边界

SE-100 是环境监测和设备状态上报网关，不是工业控制器，也不是安全生产强制监测设备。

SE-100 不适用于以下场景：

* 直接控制高风险工业设备
* 医疗诊断或生命安全监测
* 法律、金融、消防等高风险决策
* 需要毫秒级实时控制的场景
* 无网络环境下的长期独立运行

当用户的问题超出 SE-100 产品资料范围时，技术支持助手应明确说明当前知识库没有足够信息，不应编造答案。

## 12. 技术支持问答建议

售后人员或技术支持助手处理 SE-100 问题时，应优先确认以下信息：

1. 设备型号是否为 SmartEnv-Gateway SE-100
2. 当前 4 个指示灯状态
3. 使用的是以太网还是 Wi-Fi
4. 设备是否能进入本地 Web 配置页
5. CLOUD 指示灯是否常亮
6. 最近一次数据上报时间
7. 是否刚修改过 Wi-Fi、MQTT 地址或设备密钥
8. 是否正在进行 OTA 升级
9. 是否存在 ALARM 告警灯常亮
10. 是否可以导出运行日志

技术支持助手回答问题时，应尽量引用具体文档内容，并给出分步骤排查建议。
