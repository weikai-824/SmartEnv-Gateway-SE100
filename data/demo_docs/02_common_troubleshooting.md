# SmartEnv-Gateway SE-100 常见故障排查手册

## 1. 文档说明

本文档用于说明 SmartEnv-Gateway SE-100 智能环境监测网关的常见故障现象、可能原因和排查步骤。

本文档面向售后技术支持人员和普通运维用户。排查过程以指示灯状态、联网方式、本地 Web 配置页、MQTT 上报状态和运行日志为主要依据。

当用户问题超出本文档范围时，技术支持助手应说明当前知识库没有足够信息，不应编造硬件底层原因。

---

## 2. 排障前需要确认的信息

处理 SE-100 故障前，应先确认以下信息：

1. 设备型号是否为 SmartEnv-Gateway SE-100
2. 当前 PWR、NET、CLOUD、ALARM 四个指示灯状态
3. 使用的是以太网联网还是 Wi-Fi 联网
4. 设备是否能进入本地 Web 配置页
5. 最近一次数据上报时间
6. 是否刚修改过 Wi-Fi、MQTT 地址或设备密钥
7. 是否正在进行 OTA 升级
8. 是否可以导出最近运行日志

如果用户无法描述完整信息，技术支持人员应优先追问指示灯状态和联网方式。

---

## 3. 快速判断表

| 故障现象     | 常见表现            | 优先检查项                |
| -------- | --------------- | -------------------- |
| 设备完全无反应  | PWR 电源灯不亮       | 电源适配器、电源接口、插座        |
| 网络未连接    | NET 指示灯熄灭或慢闪    | 网线、Wi-Fi 密码、路由器、DHCP |
| 已联网但未连云  | NET 常亮，CLOUD 慢闪 | MQTT 地址、设备密钥、外网访问    |
| 数据不上报    | CLOUD 常亮但平台无新数据 | 上报周期、平台绑定、传感器状态      |
| 无法进入配置页  | 浏览器打不开本地地址      | 是否处于配置模式、电脑网络设置      |
| OTA 升级失败 | CLOUD 快闪后恢复原版本  | 网络稳定性、升级任务、设备重启      |
| 告警灯常亮    | ALARM 常亮        | 温湿度阈值、传感器离线、门磁状态     |

---

## 4. 故障一：设备完全无反应

### 4.1 故障现象

用户反馈设备接通电源后没有任何反应，PWR 电源灯不亮，其他指示灯也不亮。

### 4.2 可能原因

* 电源适配器未插好
* 插座无电
* 使用了非标准电源适配器
* 电源接口接触不良
* 设备硬件异常

### 4.3 排查步骤

1. 确认电源适配器是否插紧。
2. 更换插座后再次通电。
3. 确认使用的是 DC 12V 电源适配器。
4. 检查设备电源接口是否松动。
5. 若 PWR 电源灯仍不亮，建议联系售后处理。

### 4.4 处理结论

如果更换插座和电源适配器后 PWR 电源灯仍不亮，通常不建议继续自行排查，应进入售后检测流程。

---

## 5. 故障二：NET 指示灯熄灭

### 5.1 故障现象

PWR 电源灯常亮，但 NET 网络灯熄灭，设备无法连接局域网。

### 5.2 可能原因

* 网线未插好
* 路由器或交换机端口异常
* Wi-Fi 配置未完成
* Wi-Fi 名称或密码错误
* 设备未获取到 IP 地址

### 5.3 以太网排查步骤

1. 确认网线已插入 SE-100 的 LAN 接口。
2. 检查路由器或交换机端口指示灯是否正常。
3. 更换一根网线后再次测试。
4. 将网线插到其他路由器端口。
5. 重启 SE-100，等待 2 分钟后观察 NET 指示灯。

### 5.4 Wi-Fi 排查步骤

1. 确认设备是否已经进入配置模式。
2. 连接设备配置热点。
3. 进入本地 Web 配置页。
4. 重新填写 Wi-Fi 名称和密码。
5. 保存配置后等待设备重启。
6. 观察 NET 指示灯是否由慢闪变为常亮。

### 5.5 处理结论

如果 NET 指示灯持续熄灭，应优先判断是网络接入问题，而不是云端平台问题。只有 NET 指示灯常亮后，才继续排查 CLOUD 云端连接状态。

---

## 6. 故障三：NET 指示灯慢闪

### 6.1 故障现象

PWR 电源灯常亮，NET 网络灯慢闪，设备长时间无法进入正常联网状态。

### 6.2 可能原因

* DHCP 获取 IP 失败
* 路由器网络不稳定
* Wi-Fi 信号弱
* Wi-Fi 密码错误
* 局域网限制新设备接入

### 6.3 排查步骤

1. 等待 2 分钟，确认是否只是启动过程中的短暂慢闪。
2. 重启路由器后再次观察。
3. 将设备靠近路由器，排除 Wi-Fi 信号弱的问题。
4. 检查 Wi-Fi 密码是否正确。
5. 如果使用以太网，尝试更换网线和路由器端口。
6. 在本地 Web 配置页查看设备是否获取到 IP 地址。
7. 如果局域网开启了设备接入限制，需要由网络管理员放行设备。

### 6.4 处理结论

NET 指示灯慢闪通常表示设备正在尝试联网但尚未成功。排查重点应放在局域网、路由器、Wi-Fi 密码和 IP 获取过程。

---

## 7. 故障四：CLOUD 指示灯慢闪

### 7.1 故障现象

PWR 电源灯常亮，NET 网络灯常亮，但 CLOUD 云端灯慢闪。设备已经连接局域网，但没有成功连接云端平台。

### 7.2 可能原因

* MQTT 服务器地址填写错误
* MQTT 端口填写错误
* 设备密钥错误
* 当前网络不能访问外网
* 云端平台暂时不可用
* 设备时间异常导致认证失败

### 7.3 排查步骤

1. 确认 NET 指示灯是否常亮。
2. 进入本地 Web 配置页，检查 MQTT 服务器地址。
3. 检查 MQTT 端口是否填写正确。
4. 检查设备密钥是否与云端平台登记信息一致。
5. 确认当前网络可以访问外网。
6. 重启设备后等待 2 分钟。
7. 如果仍然 CLOUD 慢闪，导出运行日志交给售后分析。

### 7.4 处理结论

CLOUD 指示灯慢闪时，不应优先怀疑传感器故障。该现象通常与云端连接、MQTT 配置或外网访问有关。

---

## 8. 故障五：平台显示设备离线

### 8.1 故障现象

用户在云端平台看到设备状态为离线，或者平台长时间没有显示设备在线。

### 8.2 可能原因

* 设备断电
* 局域网断开
* MQTT 连接失败
* 设备绑定关系错误
* 云端平台未刷新
* 设备刚重启，尚未重新上线

### 8.3 排查步骤

1. 确认 PWR 电源灯是否常亮。
2. 确认 NET 网络灯是否常亮。
3. 确认 CLOUD 云端灯是否常亮。
4. 在平台查看设备绑定的序列号是否正确。
5. 等待 1 到 2 分钟后刷新平台页面。
6. 如果 CLOUD 指示灯慢闪，按“CLOUD 指示灯慢闪”流程排查。
7. 如果 CLOUD 指示灯常亮但平台仍显示离线，导出日志并联系售后。

### 8.4 处理结论

平台显示离线时，应先以设备本地指示灯为准。若 CLOUD 指示灯常亮但平台仍显示离线，可能是平台刷新、绑定关系或云端状态同步问题。

---

## 9. 故障六：数据不上报

### 9.1 故障现象

设备显示在线，但云端平台没有新的温度、湿度或门磁数据。

### 9.2 可能原因

* 数据上报周期设置过长
* 设备刚上线，尚未到上报周期
* 传感器离线
* MQTT 连接不稳定
* 平台设备绑定异常
* 云端页面筛选条件错误

### 9.3 排查步骤

1. 确认 CLOUD 指示灯是否常亮。
2. 查看平台最近一次数据上报时间。
3. 在本地 Web 配置页检查数据上报周期。
4. 确认平台页面没有设置错误的时间筛选条件。
5. 检查 ALARM 告警灯是否常亮。
6. 如果外接传感器离线，先处理传感器连接问题。
7. 重启设备后等待一个上报周期。
8. 若仍无数据，导出日志交给售后分析。

### 9.4 处理结论

数据不上报不一定等于设备离线。若 CLOUD 指示灯常亮，应优先检查上报周期、传感器状态、平台绑定和页面筛选条件。

---

## 10. 故障七：无法进入本地 Web 配置页

### 10.1 故障现象

用户连接设备配置热点后，浏览器无法打开本地 Web 配置页。

默认配置页地址为：

```text
http://192.168.4.1
```

### 10.2 可能原因

* 设备没有处于配置模式
* 用户连接的不是 SE-100 配置热点
* 电脑或手机仍在使用移动网络
* 浏览器缓存异常
* 本地地址输入错误
* 设备配置服务未启动

### 10.3 排查步骤

1. 确认设备处于配置模式。
2. 确认连接的 Wi-Fi 名称格式为 SE100-Setup-设备后四位。
3. 关闭手机移动数据后再次访问。
4. 在浏览器地址栏输入 `http://192.168.4.1`。
5. 更换浏览器后再次尝试。
6. 重启设备后重新进入配置模式。
7. 如果仍无法访问，建议恢复出厂设置后重新配置。

### 10.4 处理结论

无法进入本地 Web 配置页时，优先检查是否连接到了正确的设备配置热点，以及设备是否处于配置模式。

---

## 11. 故障八：Wi-Fi 配置后仍无法联网

### 11.1 故障现象

用户在本地 Web 配置页保存 Wi-Fi 信息后，设备重启，但 NET 指示灯仍然熄灭或慢闪。

### 11.2 可能原因

* Wi-Fi 名称填写错误
* Wi-Fi 密码填写错误
* Wi-Fi 信号弱
* 路由器限制新设备接入
* 当前 Wi-Fi 不允许设备访问外网
* 用户配置后未等待设备重启完成

### 11.3 排查步骤

1. 重新进入配置模式。
2. 检查 Wi-Fi 名称是否与路由器名称完全一致。
3. 重新输入 Wi-Fi 密码。
4. 将设备靠近路由器。
5. 重启路由器后再次配置。
6. 检查路由器是否开启了设备接入限制。
7. 保存配置后等待 2 分钟，不要立即断电。

### 11.4 处理结论

Wi-Fi 配置失败最常见原因是 Wi-Fi 密码错误、信号弱或路由器限制。排查时不要直接判断为设备硬件故障。

---

## 12. 故障九：OTA 升级失败

### 12.1 故障现象

设备收到 OTA 升级任务后，CLOUD 指示灯快闪一段时间，但升级后仍显示旧版本，或者设备重新上线后提示升级失败。

### 12.2 可能原因

* 网络连接不稳定
* 升级过程中设备断电
* 云端升级任务已过期
* 固件包下载失败
* 设备剩余存储空间不足
* 当前版本不满足升级条件

### 12.3 排查步骤

1. 确认升级过程中设备没有断电。
2. 确认 NET 和 CLOUD 指示灯状态正常。
3. 重启设备后查看当前固件版本。
4. 在云端平台重新下发升级任务。
5. 确认当前网络稳定。
6. 如果连续两次升级失败，导出日志并联系售后。

### 12.4 处理结论

OTA 升级失败后，设备通常会保留当前可用版本。不要在升级过程中断电，也不要频繁重复重启设备。

---

## 13. 故障十：ALARM 告警灯常亮

### 13.1 故障现象

设备 PWR、NET、CLOUD 状态正常，但 ALARM 告警灯常亮。

### 13.2 可能原因

* 温度超过告警阈值
* 湿度超过告警阈值
* 外接传感器离线
* 门磁状态异常
* 告警阈值设置过低
* 告警事件尚未恢复

### 13.3 排查步骤

1. 进入平台查看当前告警事件。
2. 查看温度和湿度是否超过阈值。
3. 检查本地 Web 配置页中的告警阈值设置。
4. 检查外接传感器连接是否松动。
5. 如果是门磁告警，确认门磁状态是否正常。
6. 告警条件恢复后，等待设备下一次上报。
7. 如果 ALARM 持续常亮，导出日志联系售后。

### 13.4 处理结论

ALARM 告警灯常亮不一定代表设备损坏，通常表示环境数据、传感器状态或门磁状态触发了告警条件。

---

## 14. 故障十一：外接传感器离线

### 14.1 故障现象

平台显示外接传感器离线，或者设备无法读取某一路传感器数据。

### 14.2 可能原因

* 传感器接口松动
* 传感器线缆损坏
* 传感器未正确接入 SENSOR 接口
* 传感器类型不匹配
* 设备未重新识别传感器

### 14.3 排查步骤

1. 检查传感器是否插入 SENSOR 接口。
2. 重新插拔传感器线缆。
3. 检查线缆是否破损。
4. 重启 SE-100 后查看传感器状态。
5. 如果有备用传感器，可以更换测试。
6. 若更换后仍离线，联系售后检测设备接口。

### 14.4 处理结论

外接传感器离线时，应优先检查连接线缆和传感器本身，不要直接判断为云端平台异常。

---

## 15. 恢复出厂设置

### 15.1 适用场景

以下情况可以考虑恢复出厂设置：

* 忘记 Wi-Fi 配置
* 无法进入原网络
* MQTT 参数被错误修改
* 多次配置失败
* 设备需要重新绑定

### 15.2 操作说明

1. 确认设备已经通电。
2. 长按 RESET 按键 8 秒。
3. 等待指示灯进入配置模式。
4. 连接设备配置热点。
5. 重新进入本地 Web 配置页完成配置。

### 15.3 注意事项

恢复出厂设置会清除 Wi-Fi 配置、MQTT 参数和本地告警阈值。恢复后需要重新配置设备网络和平台连接信息。

---

## 16. 需要联系售后的情况

出现以下情况时，建议联系售后人员处理：

1. PWR 电源灯始终不亮
2. 更换网线和路由器端口后 NET 仍不亮
3. CLOUD 长时间慢闪且 MQTT 参数确认无误
4. 连续两次 OTA 升级失败
5. 外接传感器更换后仍持续离线
6. 设备频繁自动重启
7. 本地 Web 配置页始终无法访问
8. 无法导出运行日志
9. 设备外壳、电源接口或传感器接口存在明显损坏

联系售后时，应提供设备型号、指示灯状态、联网方式、最近一次操作、故障截图和运行日志。

---

## 17. 技术支持回答要求

技术支持助手回答 SE-100 故障问题时，应遵守以下要求：

1. 先判断故障属于供电、联网、云端连接、数据上报、配置页、OTA、告警还是传感器问题。
2. 优先引用指示灯状态进行判断。
3. 给出分步骤排查建议。
4. 不编造本文档没有提供的硬件底层原因。
5. 如果信息不足，应先追问关键现象，例如指示灯状态、联网方式和是否能进入本地 Web 配置页。
6. 如果排查后仍无法解决，应建议用户导出日志并联系售后。
