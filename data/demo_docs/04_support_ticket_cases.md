# SmartEnv-Gateway SE-100 历史工单案例

## 1. 文档说明

本文档整理 SmartEnv-Gateway SE-100 智能环境监测网关的历史售后工单案例。每个案例包含用户反馈、现场现象、排查过程、最终结论和可复用经验。

本文档用于帮助技术支持助手学习故障排查流程、追问关键信息、生成工单摘要和给出分步骤处理建议。

所有案例均为受控虚拟案例，仅用于 Personal Knowledge Assistant 项目演示和评估。

---

## 2. 工单案例一：设备离线但 PWR 正常

### 工单编号

SE100-TICKET-2026-001

### 用户反馈

用户反馈仓储间的 SE-100 在云端平台显示离线，但设备本体 PWR 电源灯是亮的。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯慢闪
* CLOUD 云端灯熄灭
* ALARM 告警灯熄灭
* 使用 Wi-Fi 联网
* 用户最近更换过办公室路由器

### 排查过程

1. 技术支持人员先确认设备型号为 SmartEnv-Gateway SE-100。
2. 根据 NET 慢闪判断，设备尚未稳定连接局域网。
3. 继续追问用户是否更换过 Wi-Fi 名称或密码。
4. 用户确认更换路由器后 Wi-Fi 密码发生变化。
5. 技术支持人员指导用户进入配置模式。
6. 用户连接 `SE100-Setup-设备后四位` 热点。
7. 用户访问 `http://192.168.4.1` 并重新填写 Wi-Fi 名称和密码。
8. 保存配置后设备重启。
9. 约 2 分钟后，NET 和 CLOUD 指示灯均变为常亮。

### 最终结论

设备离线原因是用户更换路由器后，SE-100 仍保存旧 Wi-Fi 配置，导致无法连接局域网。

### 可复用经验

平台显示离线时，不应直接判断为云端故障。应先确认 PWR、NET、CLOUD 指示灯状态。若 NET 慢闪或熄灭，应优先排查局域网和 Wi-Fi 配置。

## 3. 工单案例二：NET 常亮但 CLOUD 慢闪

### 工单编号

SE100-TICKET-2026-002

### 用户反馈

用户反馈设备已经联网，但云端平台一直显示设备未上线。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯慢闪
* ALARM 告警灯熄灭
* 使用以太网联网
* 本地 Web 配置页可以打开
* 用户前一天修改过 MQTT 参数

### 排查过程

1. 技术支持人员根据 NET 常亮判断设备已经连接局域网。
2. 根据 CLOUD 慢闪判断设备没有成功连接云端平台。
3. 技术支持人员让用户进入本地 Web 配置页。
4. 用户检查 MQTT 服务器地址，发现地址末尾多输入了一个空格。
5. 用户删除错误空格并保存配置。
6. 设备重启后重新连接云端。
7. 约 1 分钟后 CLOUD 指示灯变为常亮。
8. 平台刷新后显示设备在线。

### 最终结论

故障原因是 MQTT 服务器地址填写错误，导致设备无法建立云端连接。

### 可复用经验

NET 常亮但 CLOUD 慢闪时，排查重点应放在 MQTT 地址、端口、设备密钥和外网访问，不应优先怀疑传感器或设备供电问题。

## 4. 工单案例三：设备在线但没有温湿度数据

### 工单编号

SE100-TICKET-2026-003

### 用户反馈

用户反馈设备在平台上显示在线，但最近半小时没有新的温湿度数据。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯常亮
* ALARM 告警灯熄灭
* 平台设备状态显示在线
* 最近一次数据上报时间停留在 30 分钟前
* 用户可以进入本地 Web 配置页

### 排查过程

1. 技术支持人员确认 CLOUD 指示灯常亮，说明设备已经连接云端。
2. 技术支持人员让用户检查数据上报周期。
3. 用户发现本地 Web 配置页中的数据上报周期被设置为 3600 秒。
4. 技术支持人员说明默认上报周期为 60 秒，3600 秒会导致平台长时间看不到新数据。
5. 用户将上报周期修改为 60 秒并保存配置。
6. 设备重启后等待 1 到 2 分钟。
7. 平台恢复显示新的温湿度数据。

### 最终结论

设备没有新数据的原因是数据上报周期被设置过长，不是设备离线，也不是传感器损坏。

### 可复用经验

设备在线但数据不上报时，应检查上报周期、最近一次上报时间、平台筛选条件和传感器状态。不要把“没有新数据”直接等同于“设备离线”。

## 5. 工单案例四：无法进入本地 Web 配置页

### 工单编号

SE100-TICKET-2026-004

### 用户反馈

用户反馈按照说明访问 `http://192.168.4.1`，但浏览器一直打不开配置页。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯快闪
* CLOUD 云端灯熄灭
* ALARM 告警灯慢闪
* 用户使用手机配置设备
* 手机连接过多个 Wi-Fi
* 手机移动数据处于开启状态

### 排查过程

1. 技术支持人员确认设备处于配置模式。
2. 技术支持人员让用户检查当前手机连接的 Wi-Fi。
3. 用户发现手机没有连接 SE-100 配置热点，而是连接到了家庭路由器。
4. 用户切换连接到 `SE100-Setup-设备后四位`。
5. 第一次访问仍失败。
6. 技术支持人员让用户关闭手机移动数据。
7. 用户重新访问 `http://192.168.4.1` 后成功打开配置页。
8. 用户完成 Wi-Fi 配置，设备重启后联网成功。

### 最终结论

无法进入配置页的原因是用户没有正确连接设备配置热点，并且手机移动数据干扰了本地访问。

### 可复用经验

无法打开本地 Web 配置页时，应优先确认用户是否连接到了 SE-100 配置热点，以及手机是否关闭移动数据。

## 6. 工单案例五：ALARM 灯常亮

### 工单编号

SE100-TICKET-2026-005

### 用户反馈

用户反馈设备可以正常联网，但 ALARM 告警灯一直亮。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯常亮
* ALARM 告警灯常亮
* 平台显示湿度告警
* 当前湿度为 78%
* 本地告警阈值设置为 70%

### 排查过程

1. 技术支持人员确认设备联网和云端连接均正常。
2. 技术支持人员让用户查看平台当前告警事件。
3. 平台显示湿度超过阈值。
4. 用户检查本地 Web 配置页，发现湿度告警阈值为 70%。
5. 当前仓储间湿度为 78%，确实触发告警条件。
6. 技术支持人员说明 ALARM 常亮表示存在告警，不代表设备损坏。
7. 用户处理仓储间湿度后，等待下一次上报。
8. 湿度降至阈值以下后，ALARM 告警状态解除。

### 最终结论

ALARM 灯常亮是因为仓储间湿度超过设定阈值，不是设备故障。

### 可复用经验

ALARM 灯常亮时，应优先查看平台告警事件和本地阈值设置。告警灯表示异常条件存在，不等于设备硬件损坏。

## 7. 工单案例六：OTA 升级失败

### 工单编号

SE100-TICKET-2026-006

### 用户反馈

用户反馈设备收到 OTA 升级任务后，CLOUD 灯快闪了一段时间，但重启后仍然显示旧版本。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯常亮
* 当前版本未变化
* 用户升级过程中曾经拔掉电源
* 设备重新上电后仍可正常联网

### 排查过程

1. 技术支持人员确认设备仍可正常联网，说明当前可用版本未损坏。
2. 技术支持人员询问升级过程中是否断电。
3. 用户确认看到指示灯闪烁后拔掉过电源。
4. 技术支持人员说明 OTA 升级过程中不能断电。
5. 用户重新发起升级任务。
6. 本次升级过程中保持供电稳定。
7. 升级完成后设备自动重启。
8. 平台显示固件版本已更新。

### 最终结论

升级失败原因是 OTA 过程中设备断电，导致升级任务未完成。

### 可复用经验

OTA 升级失败后，设备通常会保留当前可用版本。排查时应确认升级过程中是否断电、网络是否稳定，以及是否连续多次失败。

## 8. 工单案例七：外接传感器离线

### 工单编号

SE100-TICKET-2026-007

### 用户反馈

用户反馈平台显示外接传感器离线，但网关本身在线。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯常亮
* ALARM 告警灯常亮
* 平台提示外接传感器离线
* 用户近期移动过设备位置

### 排查过程

1. 技术支持人员确认 SE-100 网关本身在线。
2. 技术支持人员让用户检查 SENSOR 接口。
3. 用户发现外接传感器线缆没有插紧。
4. 用户重新插紧传感器线缆。
5. 技术支持人员指导用户重启 SE-100。
6. 设备重启后平台恢复显示外接传感器数据。
7. ALARM 告警灯熄灭。

### 最终结论

外接传感器离线原因是传感器线缆松动。

### 可复用经验

外接传感器离线时，应优先检查 SENSOR 接口、传感器线缆和传感器本身，不应直接判断为云端平台问题。

## 9. 工单案例八：平台显示离线但 CLOUD 常亮

### 工单编号

SE100-TICKET-2026-008

### 用户反馈

用户反馈平台页面显示设备离线，但现场设备 CLOUD 指示灯是常亮的。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯常亮
* ALARM 告警灯熄灭
* 用户刚刚在平台重新绑定过设备
* 平台页面未刷新

### 排查过程

1. 技术支持人员根据 CLOUD 常亮判断设备已经成功连接云端。
2. 技术支持人员让用户刷新平台页面。
3. 平台仍显示离线。
4. 技术支持人员让用户核对设备序列号。
5. 用户发现平台绑定的是另一台 SE-100 的序列号。
6. 用户删除错误绑定，并重新绑定当前设备序列号。
7. 重新绑定后平台显示设备在线。

### 最终结论

平台显示离线的原因是用户绑定了错误的设备序列号。

### 可复用经验

如果 CLOUD 指示灯常亮但平台显示离线，应检查平台刷新状态和设备绑定关系。此时不应优先排查 Wi-Fi 或 MQTT 连接。

## 10. 工单案例九：设备频繁自动重启

### 工单编号

SE100-TICKET-2026-009

### 用户反馈

用户反馈设备每隔几分钟自动重启一次，平台在线状态反复变化。

### 已知现象

* PWR 电源灯会周期性熄灭后重新亮起
* NET 和 CLOUD 状态反复变化
* 用户使用了非原装电源适配器
* 设备安装在仓储间插排上

### 排查过程

1. 技术支持人员注意到 PWR 灯会周期性熄灭。
2. 技术支持人员询问电源适配器规格。
3. 用户确认使用的是临时替换的非标准电源适配器。
4. 技术支持人员建议更换为 DC 12V 电源适配器。
5. 用户更换电源适配器后，设备不再频繁重启。
6. 平台在线状态恢复稳定。

### 最终结论

设备频繁重启与供电不稳定有关，主要原因是使用了非标准电源适配器。

### 可复用经验

设备频繁重启时，应优先确认供电稳定性和电源适配器规格，尤其要观察 PWR 电源灯是否周期性熄灭。

## 11. 工单案例十：恢复出厂设置后无法上云

### 工单编号

SE100-TICKET-2026-010

### 用户反馈

用户反馈恢复出厂设置后，设备可以连上 Wi-Fi，但无法连接云端。

### 已知现象

* PWR 电源灯常亮
* NET 网络灯常亮
* CLOUD 云端灯慢闪
* 用户刚执行过恢复出厂设置
* Wi-Fi 配置已经重新填写
* MQTT 参数没有重新填写

### 排查过程

1. 技术支持人员根据 NET 常亮确认设备已经连上局域网。
2. 根据 CLOUD 慢闪判断设备没有成功连接云端。
3. 技术支持人员询问是否重新填写 MQTT 参数和设备密钥。
4. 用户表示只填写了 Wi-Fi，没有填写 MQTT 信息。
5. 技术支持人员说明恢复出厂设置会清除 Wi-Fi 配置、MQTT 参数和本地告警阈值。
6. 用户重新填写 MQTT 地址、端口和设备密钥。
7. 保存配置后设备重启。
8. CLOUD 指示灯变为常亮，设备恢复上线。

### 最终结论

恢复出厂设置后无法上云，是因为 MQTT 参数和设备密钥被清除但没有重新配置。

### 可复用经验

恢复出厂设置后，不能只重新配置 Wi-Fi，还需要重新配置 MQTT 地址、端口、设备密钥和必要的告警阈值。

## 12. 工单摘要生成要求

技术支持助手生成工单摘要时，应包含以下信息：

1. 用户反馈的问题。
2. 当前指示灯状态。
3. 联网方式。
4. 已确认的关键事实。
5. 排查步骤。
6. 最终原因。
7. 处理结果。
8. 是否需要售后继续跟进。

示例摘要：

用户反馈 SE-100 在平台显示离线。现场 PWR 常亮，NET 慢闪，CLOUD 熄灭，使用 Wi-Fi 联网。排查发现用户更换路由器后 Wi-Fi 密码发生变化，设备仍保存旧 Wi-Fi 配置。指导用户进入配置模式并重新填写 Wi-Fi 信息后，NET 和 CLOUD 指示灯恢复常亮，平台显示设备在线。本工单无需继续售后跟进。

## 13. 轻量 Agent 追问建议

当用户问题信息不足时，技术支持助手可以按以下顺序追问：

1. 请问设备型号是否为 SmartEnv-Gateway SE-100？
2. 当前 PWR、NET、CLOUD、ALARM 四个指示灯分别是什么状态？
3. 设备使用的是以太网还是 Wi-Fi？
4. 平台显示的是离线、无数据，还是有告警？
5. 最近是否修改过 Wi-Fi、MQTT 地址、设备密钥或执行过恢复出厂设置？
6. 是否可以进入本地 Web 配置页？
7. 是否可以导出运行日志？

追问时应一次只问必要信息，不要一次性让用户填写过多内容。
